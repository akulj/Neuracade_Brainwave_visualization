Each channel renders from its own ring buffer on a throttled draw loop, decoupled from the serial read loop — this is what keeps this tab smooth even while FFT/SFDR run in a background worker. Double-click a plot to export that channel's visible window as CSV.

Analysis runs continuously on the selected channels in a Web Worker (Welch PSD, 50% overlap, Hann window) — a true non-realtime FFT batch every interval, which avoids the single-shot non-real-time approach in fft_check.py while still not blocking the UI thread.

Uses the same background analysis as the Spectrum tab. Mu (8–13 Hz) and beta (13–30 Hz) power are the bands most associated with sensorimotor rhythm modulation during motor imagery (event-related desynchronization). Rename channels below (e.g. to C3 / C4) — the mapping is saved locally.

Left / Right mu-power asymmetry

—

Asymmetry = (left − right) / (left + right). Negative values trend toward right-hand imagery desynchronizing C3 more than C4, and vice versa — a common bare-bones motor imagery feature; treat as a starting point, not a validated classifier.

Detects movement bursts against a self-adapting noise floor instead of a fixed threshold, so the slowly drifting baseline you're seeing can't cause false triggers or missed detections. Ratio = burst energy ÷ recent quiet-baseline energy — it stays near 1 at rest no matter where the raw signal currently sits. Absolute thresholds are hardware/session specific, so calibrate below rather than trusting the defaults: stay still for the baseline step, then perform each movement once, cleanly, when prompted.
